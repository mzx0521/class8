# 作品展示页面需求文档

## 1. 应用概述

### 1.1 应用名称
作品展示页面

### 1.2 应用描述
这是一个用于展示个人作品和学习成果的网页应用,作为课程毕业名片。页面包含项目路演、学习复盘、对话机器人等内容模块,展示发现的痛点场景、MVP产品、学习心得与收获,并提供代码仓库和公网链接信息。

## 2. 核心内容模块

### 2.1 项目路演

#### 2.1.1 痛点场景
- 场景描述:需要针对特定会议场景的深度辅助助手,像军师一样陪开会,给出建议
- 场景来源:工作过程中需求调研会、需求分析会、需求交底会等会议内容不完整、遗漏的问题
- 场景需求:会前了解会议资料和角色,会中实时记录、分析、查缺补漏,会后生成会议纪要或专业报告

#### 2.1.2 点子到产品的思路
- 第一阶段:找场景
  - 如何找场景:从工作生活中面临的痛点、爽点和痒点出发,头脑风暴多个点子
  - 如何选择场景:选择值得长期做且有价值的场景,通过与周围同事验证确认价值
- 第二阶段:定方案
  - 调研分析:搜索类似产品(飞书会议付费版),与AI共创产品命名、定位、技术可行性、版本规划等
  - 开始干活:形成分析报告,给claude code对齐需求、确定技术可行性、规划版本,形成PRD
- 第三阶段:磨产品
  - MVP开发过程:十几分钟完成开发,但解决问题超过6小时,面临各种问题需要不断与AI工具磨产品

#### 2.1.3 MVP产品方案
- 目标:主要针对网页版会议
- 形式:浏览器插件
- 音频捕捉方案:chrome.tabCapture
- ASR方案:阿里云通义听悟+deepseek
- 会议记录方案:docxtemplater导出成docx格式文件

#### 2.1.4 MVP产品功能
- 浏览器插件,在网页端运行
- 支持在线捕捉音频
- 支持音频转文字,并定期10段对话总结一次
- 支持基于阶段总结生成针对性建议(实时军师在线)
- 支持会议纪要整理生成
- 支持docx格式纪要下载

#### 2.1.5 MVP产品展示
- 安装插件
- 打开服务端
- 打开插件
- 启动军师
- 给出建议(示例)
- 下载纪要(示例)

### 2.2 学习复盘

#### 2.2.1 心法篇
- 顺势而为,得势者胜:抓住AI coding行业趋势,AI coding将成为AI时代的元能力
- 行动起来,不要等待:先动手做起来,边做边学,不要等待完全学会再行动
- 产品思维,工程思维:区分产品思维和工程思维,面向用户用产品思维,凡工程皆产品
- 别把自己当学生,把自己当老板:AI是24小时待命的顶级程序员,只需说明需求即可
- 不怕报错,就怕没逻辑:代码跑不通是正常现象,关键是逻辑要清楚
- 先做垃圾,逐渐完美(MVP意识):先做一个核心功能的小破车,能跑起来就有信心
- AI思维:万事不决问AI,但要保持自己的判断力
- 拆解思维:遇到大项目和大问题要学会拆解成小部分逐一解决

#### 2.2.2 方法篇
如何通过AI coding制作出产品?
1. 想清楚到底要啥(明确需求):说清楚具体功能和预期效果
2. 让AI先试水(快速Demo验证):先让AI甩出最简单代码,验证可行性
3. 商量怎么干(讨论可行性与版本规划):询问AI功能难度,规划版本
4. 写说明书(写PRD):把想法写成文档喂给AI,避免跑偏
5. 动手盖房子(UI设计与开发):
   - 搞装修:用自然语言告诉AI设计要求
   - 搭框架:在Cursor里下指令让AI码功能
6. 找茬儿与上线(测试与发布):自己测试没问题后一键发布到网上

这个过程是从简单到复杂、迭代优化提升的过程。

#### 2.2.3 收获篇
我学会了什么?
- 思维升级:用产品思维解构产品
- 产品流程:从痛点到MVP的全流程
- 需求分析:写PRD文档
- 开发技能:使用enter.pro、claude code开发工具或产品
- 数据接入:接入Supabase数据库、API接口、各类模型
- 发布部署:发布到enter.pro、vercel等服务器
- 埋点分析:接入Google Analytics、PostHog等数据分析工具
- SEO优化:进行SEO装修优化
- 域名配置:购买域名、配置域名搭建独有网站
- Skills创建:创建skills和使用skills提效
- 插件开发:开发浏览器插件
